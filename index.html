SELFI3-D: Facial Mesh Reconstruction and Tattoo Visualization from Sparse Input Views

Team Members: Akshaan Ahuja, Michelle Chen, Jameson Crate, Valerie Li

Summary and Team Members

We plan to use gaussian splatting, poisson reconstruction and texture mapping to create realistic renderings of face tattoos. Upon creating the mesh of the face, we will overlay a texture map to render a realistic facial appearance. Adding onto the base texture, we will overlay a second texture map that contains the tattoo design.

Problem Description

Lets be honest, everyone these days wants a face tattoo, but most of them come out looking horrible. We present SELFI3-D, a pipeline for facial mesh reconstruction and tattoo texture mapping that will help convince worried parents that a face tattoo is the right decision before unreversable steps are taken. Our problem addresses the problem of reconstructing faces from a sets of images with realistic texture and structure. Additionally we take arbitrary 2D images as texture maps in order to visualize the image mapped onto an individual's face. We will use gaussian splatting, poisson reconstruction and mesh regularization methods to reconstruct a mesh of a face. We will then use texture mapping to map the indivudal's facial features onto the mesh as well as the selected tattoo. The challenge lies in achieving a precise mesh reconstruction that will render a realistic huminoid picture!

Goals and Deliverables

What We Plan to Deliver (Baseline)

Use sparse views of an individual's face to create a 3D facial reconstruction with textured mapped tattoos using gaussian splatting and poisson reconstruction.

  • Use 3D Reconstruction to create a rendering of a person's face.
  • Using the output mesh, implement a loop subdivision scheme to upscale the mesh as well as texture map the mesh to look like the input person.
  • Apply a texture mapping of desired face tattoo to visualize it on the person's face.

What We Hope to Deliver (Aspirational)

If we are ahead of schedule and things are going well, we wish to involve animation. By creating an internal skeleton corresponding to the facial features (rigging), we can then animate the face and have it speak or show expressions. Additionally, we can add custom scene lighting to make our rendered photos look nice or view tattoos under different lighting conditions such as in the clurb. We also want to expiriment with dynamic tattoos.

  • Create a rig for the mesh to animate the Face
  • Expression transfer between different faces
  • Dynamic Tattoos (using gifs as tattoo texture maps)

How We Will Measure Success

We will measure reconstruction accuracy via the Chamfer Distance or Hausdorff Distance, which will compare the reconstructed mesh with a ground-truth facial image. Graph: Bar or line chart comparing reconstruction error across different methods (e.g., raw vs. loop-subdivided mesh) We will also measure the SSIM (Structural Similarity Index) or PSNR (Peak Signal-to-Noise Ratio) which will evaluate how well the texture-mapped face resembles the original image. We will show side-by-side images of the original image and the textured mesh. Graph: SSIM/PSNR scores for various stages of texture mapping (base mesh vs. subdivided vs. final output).

Questions we plan to answer:

  • How well can we apply a custom texture mapping to a generated mesh of a face?
  • How well can texture mapping mimic the nuances of a human's face?
  • What would I look like with a sick face tattoo?
